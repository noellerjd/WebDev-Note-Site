Introduction to CSS

Date: 2/3/24

Element Selectors: allows you to apply styles to all of a certain tag in your html.
Such as p{font-size:3rem;}

Attribute Selectors: selects elements with a certain attribute. p[draggable]{color:blue;} will only effect p tags with the draggable attribute.
You can also select their value inversely by doing something like p[draggable="false"]{color:blue;}, which will only effect p tags that have their draggable attribute set to false.

Universal Selector: * using an astrisk you can select everything and apply a style universally.
Such as * {font-family: "Times New Roman", serif;}

ID Selector: allows you to style a specific element with a set ID. Usually this is for elements that won't be used more than once, such as a header or footer, etc. It will look something like this
#header{background-color:white;}

Class Selector: allows you to style a specific element with a set class. Usually this is for elements that you will be repeating, so you can apply style to the class that will automatically apply to everything with this class, such as wanting to apply underlines to every p with the class="underlined" and would look like this:
.underlined{font-decoration: underline;}

Font Properties

Date: 2/3/24

Font Size

1px: meaning 1 pixel or 1/96th and inch, about the size of a pixel.

1pt: meaning 1 point or 1/72nd of an inch. Typically used in word processors like Microsoft Word or Gmail/Outlook.

1em: literally meaning the size of an M, the widest letter in the alphabet. 1m would be 100% of the parent.

1rem: meaning root em. This would be relative to the root of the HTML instead of the parent. 1rem = ~20px

Text Align

**Apologies if this throws you off, I styled each to represent how they would look in actual code.**

Center: centers text in the middle of their parent.

Left: aligns text to the left of their parent.

Right: aligns text to the right of their parent.

Start: aligns text to the start of their parent, depending on the language it's coded. Left in english.

End: aligns text to the end of their parent, depending on the language it's coded. Right for english.

Justify: lines up text to align the structure of text to be the starting edge and the ending edge, and should space the text dynamically to make this happen for everything except the last word.

Intermediate CSS

Date: 2/4/24

Cascading Style Sheets

Position: CSS will read top to bottom, so one selector at the bottom will be the final one applied, for example if you have two attribute selectors both referencing line items:

li {color:red} <--- this one is read first

li {color:blue} <--- this one is read second.

The last one to be read will be applied, therfore the line items would be blue.

Specificity: CSS will prioritize specifity over position, for example any style that refrences a class will override any element selectors. Any ID selectors will take priority over class selectors. Any attribute selectors will also override class selectors but won't override ID selectors. See image below:

Type: CSS will prioritize linked stylesheets last. Higher on priority is the HTML contained CSS (the styling you do in the <head> of the html document). The highest priority is styling done withing the physical tags of the HTML such as <li style="color:red">

!Important: this is a keyword you can apply to any rule ie. p{color:green !important;} this will give the styling highest priority no matter what.

Alternate Selectors

Grouping Selectors: h1, h2{color:blueviolet;} will group two different selectors with the same styling. In this case both h1s and h2s will become blueviolet.

Child Selectors: .box > p {color:firebrick} can be used to select a direct child of an element. In this case any p tags that are children of something with the "box" class will change their color to firebrick

Descendant Selector: .box li{color:blue} or selector selector{color:blue}. These will apply css to the descendant of the left selector, in example the li that is a sescendant of something with the "box" class.

Chaining Selector: SelectorSelector{color:seagreen} or h1#tile.big.heading{color:seagreen;}. A chaining selector is a very specific way to apply styles to an element, there should be no spaces when using this selector and element selectors must always go first.

Combining Combiners: Selector SelectorSelector{font-size:0.5rem;} or ul p.done{font-size:0.5rem}. This is a way to select a more specific element for styling in css. In the example, we are selecting a p element with the class of done but only if it's a descendant of a ul element.

CSS Positioning

There are four different values for positioning.

Static Positioning: HTML's default flow. item flows as it would in the actual html file. Displayed below the code above it by default.

Relative Positioning: Item positioned relative to it's default position. Changing the items position would be relative to it's supposed position.

Absolute Positioning: Sets item's position to nearest positioned anscestor, or top left corner of the webpage. If the ancestor doesn't have a position set, the second rule becomes true.

Fixed Positioning: Positions relative to the top left corner of the browser. This won't change even when scrolling up or down on the webpage and will always remain visable by default. This is what I used to have the menu on desktop stay no matter where the user scrolls in this webpage.

- a lot of the time you will see a parent set to relative and the child set to absolute

Display Properties

Block: display:block; Default formatting, if you have another tag or element it'll go below the item as normal.

Inline: display:inline; With inline, if you have two tags it'll make them go side by side until there's no more room in the width of the browser. Inline elements will default to the width of their contetns.

Inline Block: display:inline-block; The block portion of this display property allows us to set the height and width of the item. The inline portion allows us to have elements that go on the same line.

None: display: none; This will make the element dissapear from the webpage.

CSS Float

"wrapping text using float and clear"

Float Property: float:left; float:right; This allows elements (such as an image) to be take out of the normal flow of the webpage and wraps the rest of the page around them.

Clear Property: clear:left; clear:right; clear:both; Allows elements to be exempt from other floating elements ie. used to exclude a footer from a floating element to keep the footer at the bottom of the page.

Responsive Websites

Using tools to make websites functional on ALL devices.

Media Query: @media(max-width:600px){*css for screens 600px wide or less*} can be used to change styling for users on different width devices like tablets and phones.

Min-width will do the inverse of max width. You can also combine breakpoints to target screen sizes more specifically, ie. @media(min-width:600px) and (max-width:900px){color:blue} in this example screens that are 600px or higher and 900px or lower will have the text color be blue. This is used in order to design styles for a specific screen size, like a tablet or phone. You can also inverse the min and max to exclude styling on a specific screen size.

You can also specify styles for different screen orientations, ie. @media screen(orientation:landscape){color:red} to change styles for only landscape or portrait modes.

Typical Screen Size Ranges:

  • Mobile Devices: 319px - 480px
  • Tablets: 481px - 1200px
  • Laptops: 1200px - 1600px
  • Desktops: 1600px+

CSS Grid: You can use a grid format to complete the layout by creating a div with a class like "grid-container" followed by specific card layouts within the container and set the display to display:grid;

You can then use grid-template-columns: 1fr 1fr; for example to specify two columns. 1fr meaning 1 fraction, splitting these columns into two rows of the screen.

You can then use grid-template-rows similarly to specify how many rows to display. This can be done in the same way with 1fr or you can alternatively set a pixel height, etc.

Using this method you can also set a gap between the rows and columns by using gap: 30px; for example, to specify a 30px gap between each row.

If you wanted one column to span two rows (for a header or larger column for example) you can make the class something like: "first card" so it'll still retain the card class but then with .first you can specify grid-column:span 2; or grid-row:span 2; in order to expand the card with the class .first to be the width of two columns or rows, etc.

CSS Flexbox Similar to the grid system which is good for 2D design, flexbox is good for 1D design; changing items to apply to vertical or horizontal styling.

You can use display:flex; on a parent div to then apply flex:1; to all children to let all flexible children be the same length regardless of its content. You can alternatively change the distribution of each child by changing their flex value.

This would still be similar in some ways to the grid style, using containers and cards, but would be more visually complex and responsive.

Bootstrap Frameworks: This is a method that uses frameworks pre-coded by other programmers to cut the css down and has different rules for different selectors that are applied by putting in different key words into your classes.

Bootstrap for example also has it's own system for putting in responsive design, ie. class="col-4".

Typically the sight is divided into 12 sections and by using col-4 it will take up 4/12 of a section, or 1/3 of the webpage. They will respond based on how much of the fraction of the webpage they take up.

CSS Flexbox

Date: 2/6/24

Flex Box

Display:flex; is completely different from inline, block, etc. It no longer abides by these rules once used. When we give an item display:flex; we give some control to flexbox to allow it to display the code in a reasonable and commonly desired way. With flexbox we have a lot of modifiers to tweak it to achive the look we're going for

Flex box ignores any pre-defined rule (display:block; display:inline; etc.) and instead wants to put everything together based on the content within the element to fit them. That being said you can still use the gap:30px for example to put space between the elements.

Display: inline-flex; this will try to make your flexbox able to be next to another element or flexbox; similar to a wider reaching inline. This will force the elements to only take up as much width as the widest tag, it's the same as flex but horizontally.

Flex Direction

By default the flex direction is set to: row. So the items within the flexbox will align horizonally. When set to row, this is called the Main Axis. Alternatively, the opposite axis is called the Cross Axis. If the flex direction is set to row, the cross axis would be vertical. The opposite is true if the flex direction is set to: column. ie. flex-direction: column; flex-direction: row;
You can also set them to be in reverse, ie. flex-direction: row-reverse; and flex-direction: column-reverse;

Flex-basis: The flex basis will expand the flex box along the main axis to satisfy the flex-basis requirement. Up and down if direction is set to column, left and right if direction is set to row. For this to work, flex-basis must be set on the children, not the parent.

Flexible Layout

flex-wrap: by default if the flexbox runs out of room it wont automatically move to the next line. You can change this from it's default flex-wrap: nowrap; to flex-wrap: wrap; to have it move items without room to the next line. This is a property that is set on the parent, not the child.

flex-flow: is a way to combine both the flex-direction and flex-wrap in one line as they are often used together. ie. flex-flow: row wrap;

Order: with a flexbox, by default everything has an order of 0, you can adjust an item and set it's order:1; for example and if everything else is still at the default "0", then the set item will appear last in the flexbox. You can set order:" "; to integer and it must be set on the child, not the parent.

justify-content: sets the distribution of items within a flexbox along the main axis. you can use justify-content: flex-start to have the items line up from the left or the start of the flexbox, inversely you can use justify-content: flex-end; for the opposite effect (as long as there's room within the flexbox to change this) justify-content: center will make the items gravitate toward the center of the flex box. justify-content: space-between will allow you to have each item spaced to take up the entire width of the flexbox-- adding gaps between items to make this happen. justify-content: space-evenly will line the items up to make the items line up evenly from the center of the flexbox to take up the width of the flexbox. justify-conter: space-around will distribute items evenly and make the start and end gaps half the size of the space between each item.

Align Items: is very similar to justify-content except it will alter the items on the cross-axis. Unlike justify-content, for align-items to work you'll need to set the height of the container, and works best if the flex-wrap is set to nowrap. You can still use similar values for the styling as you did with justify-content such as, flex-start; flex-end; center; but can also use align-items: stretch; to have the items take up the entire cross axis of the flex-box. This is set on the parent, not the child.

align-self: allows you to set the alignment of a singular item within the flexbox by assigning it on the child you want to affect. You can still use similar values for the styling as you did with justify-content and align-items such as, flex-start; flex-end; center;

align-content: align-content is similar to align items except it's made to work with flexbox's that are set to wrap, instead of nowrap. You can still use similar values for the styling as you did with justify-content and align-items such as, flex-start; flex-end; center; This is set on the parent, not the child.
