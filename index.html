Introduction to CSS

Date: 2/3/24

Element Selectors: allows you to apply styles to all of a certain tag in your html.
Such as p{font-size:3rem;}

Attribute Selectors: selects elements with a certain attribute. p[draggable]{color:blue;} will only effect p tags with the draggable attribute.
You can also select their value inversely by doing something like p[draggable="false"]{color:blue;}, which will only effect p tags that have their draggable attribute set to false.

Universal Selector: * using an astrisk you can select everything and apply a style universally.
Such as * {font-family: "Times New Roman", serif;}

ID Selector: allows you to style a specific element with a set ID. Usually this is for elements that won't be used more than once, such as a header or footer, etc. It will look something like this
#header{background-color:white;}

Class Selector: allows you to style a specific element with a set class. Usually this is for elements that you will be repeating, so you can apply style to the class that will automatically apply to everything with this class, such as wanting to apply underlines to every p with the class="underlined" and would look like this:
.underlined{font-decoration: underline;}

Font Properties

Date: 2/3/24

Font Size

1px: meaning 1 pixel or 1/96th and inch, about the size of a pixel.

1pt: meaning 1 point or 1/72nd of an inch. Typically used in word processors like Microsoft Word or Gmail/Outlook.

1em: literally meaning the size of an M, the widest letter in the alphabet. 1m would be 100% of the parent.

1rem: meaning root em. This would be relative to the root of the HTML instead of the parent. 1rem = ~20px

Text Align

**Apologies if this throws you off, I styled each to represent how they would look in actual code.**

Center: centers text in the middle of their parent.

Left: aligns text to the left of their parent.

Right: aligns text to the right of their parent.

Start: aligns text to the start of their parent, depending on the language it's coded. Left in english.

End: aligns text to the end of their parent, depending on the language it's coded. Right for english.

Justify: lines up text to align the structure of text to be the starting edge and the ending edge, and should space the text dynamically to make this happen for everything except the last word.

Intermediate CSS

Date: 2/4/24

Cascading Style Sheets

Position: CSS will read top to bottom, so one selector at the bottom will be the final one applied, for example if you have two attribute selectors both referencing line items:

li {color:red} <--- this one is read first

li {color:blue} <--- this one is read second.

The last one to be read will be applied, therfore the line items would be blue.

Specificity: CSS will prioritize specifity over position, for example any style that refrences a class will override any element selectors. Any ID selectors will take priority over class selectors. Any attribute selectors will also override class selectors but won't override ID selectors. See image below:

Type: CSS will prioritize linked stylesheets last. Higher on priority is the HTML contained CSS (the styling you do in the <head> of the html document). The highest priority is styling done withing the physical tags of the HTML such as <li style="color:red">

!Important: this is a keyword you can apply to any rule ie. p{color:green !important;} this will give the styling highest priority no matter what.

Alternate Selectors

Grouping Selectors: h1, h2{color:blueviolet;} will group two different selectors with the same styling. In this case both h1s and h2s will become blueviolet.

Child Selectors: .box > p {color:firebrick} can be used to select a direct child of an element. In this case any p tags that are children of something with the "box" class will change their color to firebrick

Descendant Selector: .box li{color:blue} or selector selector{color:blue}. These will apply css to the descendant of the left selector, in example the li that is a sescendant of something with the "box" class.

Chaining Selector: SelectorSelector{color:seagreen} or h1#tile.big.heading{color:seagreen;}. A chaining selector is a very specific way to apply styles to an element, there should be no spaces when using this selector and element selectors must always go first.

Combining Combiners: Selector SelectorSelector{font-size:0.5rem;} or ul p.done{font-size:0.5rem}. This is a way to select a more specific element for styling in css. In the example, we are selecting a p element with the class of done but only if it's a descendant of a ul element.

CSS Positioning

There are four different values for positioning.

Static Positioning: HTML's default flow. item flows as it would in the actual html file. Displayed below the code above it by default.

Relative Positioning: Item positioned relative to it's default position. Changing the items position would be relative to it's supposed position.

Absolute Positioning: Sets item's position to nearest positioned anscestor, or top left corner of the webpage. If the ancestor doesn't have a position set, the second rule becomes true.

Fixed Positioning: Positions relative to the top left corner of the browser. This won't change even when scrolling up or down on the webpage and will always remain visable by default. This is what I used to have the menu on desktop stay no matter where the user scrolls in this webpage.

- a lot of the time you will see a parent set to relative and the child set to absolute
